### Visual Hierarchy

**Auth screens (login/register):** The brand name "RecipeHub" in 24px serif semibold at the top-center of the form card is the primary focal point — it anchors the page and establishes the editorial tone. The form itself is contained in a bordered white card (max-width 400px, centered) that floats on the warm off-white background. The eye moves: brand → heading → first form field label (11px mono uppercase) → input → CTA button (dark bg, white text, slight shadow offset). Error messages interrupt this flow with a red-bordered alert container above the form fields.

**404 page:** The oversized "404" numeral (60px+ serif bold) dominates the viewport center. The rotated SearchX icon in a square container above it provides a secondary anchor. The "Return to Feed" CTA is the tertiary focal point at the bottom of the card.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 15px (text-[15px] / sm:text-base) | 400 (regular) | 1.6 |
| Label | 11px (text-[11px]) | 600 (semibold) | 1.3 |
| Heading | 24px (text-2xl) | 600 (semibold) | 1.2 |

**Font families by role:**
- Body: `IBM Plex Sans` (--font-sans) — paragraph text, input values, links, helper text
- Label: `IBM Plex Mono` (--font-mono) — form field labels, buttons, nav items, error headings, all in `uppercase tracking-widest`
- Heading: `IBM Plex Serif` (--font-serif) — page titles, brand name, auth page headings, with `tracking-tight`

**Source:** `frontend-to-implement/src/index.css` body rule (15px, 1.6 line-height). Auth pages (Login.tsx, Register.tsx) use text-[11px] font-mono for labels and font-serif text-2xl for headings. Google Fonts import loads weights 400, 500, 600, 700 for all three families — the contract uses 400 (regular) and 600 (semibold) exclusively.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #f5f4f0 (--color-bg) | Page background (`body`, `min-h-screen`) |
| Secondary (30%) | #ffffff (--color-surface) | Form cards, header nav bar, footer |
| Accent (10%) | #c0392b (--color-accent) | Reserved for: brand name hover state, link underline hover, inline link hover transitions |
| Destructive | #e74c3c (--color-danger) | Error message containers, error text, inline validation error indicators |

**Supporting colors (not in 60/30/10 split):**

| Token | Value | Usage |
|-------|-------|-------|
| --color-text-custom | #1a1a1a | Primary text, button backgrounds (dark), form borders on focus, active nav underline |
| --color-text-muted | #6b6b6b | Secondary text, inactive nav links, footer text |
| --color-border-custom | #d0cec8 | Form input borders, card borders, header/footer borders, divider lines |
| --color-accent-hover | #a93226 | Accent hover darkening (available, not heavily used in Phase 1) |
| --color-star | #e67e22 | Star ratings (Phase 2 — declared here for completeness) |

**Accent reserved for:** Brand name hover (`hover:text-accent` on "RecipeHub" logo), link hover underline decoration (`hover:border-accent` on register/sign-in links).  
**NOT used for:** Primary buttons (these use `bg-text-custom` / `#1a1a1a` — dark), form borders, nav active states, or general interactive elements.  
**Destructive reserved for:** Error alert containers (`bg-red-50 border-danger text-danger`), inline validation error text. No destructive actions (delete, remove) exist in Phase 1.

**Source:** `frontend-to-implement/src/index.css` @theme block (lines 4-17). Color usage patterns verified against Login.tsx (lines 79-83 for error, 120 for primary button), Register.tsx (same pattern), Header.tsx (line 62 for brand hover accent).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (login) | **Sign in** — button text on `/login` form. Loading state: "Authenticating..." while submitting. |
| Primary CTA (register) | **Create account** — button text on `/register` form. Loading state: "Creating account..." while submitting. |
| Login heading | "Sign in to your account" — h2 below brand name |
| Register heading | "Create your account" — h2 below brand name |
| Login link | "Don't have an account?" → **Create account** — link with ArrowRight icon to /register |
| Register link | "Already have an account?" → **Sign in** — link with ArrowRight icon to /login |
| Auth error heading (login) | "AUTH EXCEPTION" — mono uppercase label in error container |
| Auth error heading (register) | "REGISTRATION EXCEPTION" — mono uppercase label in error container |
| Error state (login) | Server returns: "Invalid email or password." — display verbatim. Do NOT differentiate "email not found" vs "wrong password" (prevents enumeration). |
| Error state (register) | Display backend error message verbatim (e.g., "Email is already registered"). Zod client-side errors: field-level messages below each input. |
| 404 page heading | "404" — large serif display. Subtitle: "Sector Not Found" — mono uppercase. |
| 404 page body | "The coordinates you provided do not correspond to any active recipe or catalog entry in our database." |
| 404 page CTA | **Return to Feed** — button with ArrowLeft icon linking to `/` |
| Sign out button | **Sign out** — mono uppercase button in header, visible only when authenticated. Next to user greeting: "Hello, [name]". |
| Sign in button (header) | **Sign in** — mono uppercase button in header, visible only when unauthenticated. Links to `/login`. |
| Destructive confirmation | None in Phase 1 — no delete, remove, or destructive actions exist in this phase. |

**Source:** All copy extracted from existing scaffold pages: Login.tsx, Register.tsx, NotFound.tsx, Header.tsx. Phase 1 adapts these pages to real backend — copy remains consistent.
