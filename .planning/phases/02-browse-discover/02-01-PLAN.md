## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|----------------|
| T-02-01 | Spoofing | Home.tsx fetchRecipes | accept | Recipe data integrity depends on backend auth — no frontend spoofing risk for read-only feed |
| T-02-02 | Tampering | RecipeCard.tsx image onError | mitigate | Image `onError` handler sets `imgError` state to show placeholder — prevents broken image injection. `referrerPolicy="no-referrer"` on img tags prevents referrer leakage. |
| T-02-03 | Information Disclosure | RecipeCard.tsx comment fetch | accept | GET /api/recipes/:id/comments returns public data — no PII exposure. Rating averages are aggregate data. |
| T-02-04 | Denial of Service | Home.tsx recipe fetch | accept | Backend rate-limiting is the correct mitigation layer — frontend has no additional DoS surface beyond normal HTTP requests |
| T-02-05 | Elevation of Privilege | N/A (this plan) | accept | No auth-gated operations in this plan — feed and card view are public |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck + `npx shadcn@latest add badge skeleton` uses official shadcn registry (verified in 02-UI-SPEC.md Registry Safety) |
</threat_model>
