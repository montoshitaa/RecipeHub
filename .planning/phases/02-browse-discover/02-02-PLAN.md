Search query `q` param is sanitized by `useSearchParams` (URL-encoded) and client-side `.toLowerCase().trim()` filtering prevents injection. |

Additional guard: text search is client-side only (filters `recipes[]` in memory), no API call on every keystroke. Category/difficulty changes trigger API re-fetch but dropdown interaction is low-frequency. |
