### Visual Hierarchy

**Recipe feed (home page):** The editorial hero block ("Recipe Catalog" in serif display) anchors the top of the page. Below it, a horizontal filter bar (search input, category select, difficulty select) spans the full width. The recipe card grid (1-col mobile, 2-col tablet, 3-col desktop) fills the remaining viewport. Each card follows: image → category/difficulty metadata row → title → time/servings pills → star rating + "View" CTA.

**Recipe detail page:** Two-column layout (60/40 on desktop, stacked on mobile). Left: hero image (16:9, max-h 360px) → serif title → italic description with accent left-border → metadata summary bar (category, difficulty, time, servings) → preparation steps (numbered, bordered). Right: author card → ingredients table → comment section (two-column: comments list + comment form). Back-to-catalog link and owner edit/delete controls in top bar.

**Difficulty badges:** Color-coded pill badges replace plain text difficulty display. Easy=green (#27ae60), Medium=amber (#e67e22), Hard=red (#c0392b). Appear on recipe cards (top-right of image area, or inline in metadata row) and in the detail page metadata bar.

**Ingredient checklist:** Each ingredient row in the detail page ingredients table renders with a checkbox on the left. Checking off an ingredient applies a strikethrough + muted opacity. State is local only (persisted in `useState`, resets on page leave — no backend sync). UX-07.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #f5f4f0 (--color-bg) | Page background (`body`, `min-h-screen`) |
| Secondary (30%) | #ffffff (--color-surface) | Cards, filter bars, detail page content blocks, comment form, header nav |
| Accent (10%) | #c0392b (--color-accent) | Reserved for: card title hover state, hard difficulty badge, detail page description left-border, star rating AVG display, interactive link hover |
| Destructive | #e74c3c (--color-danger) | Delete comment button, delete recipe confirmation, error alert containers |

**Accent reserved for:** Card title hover (`group-hover:text-accent`), Hard difficulty badge background, description left-border (`border-l-2 border-accent`), average star rating numeric display, "View" link hover underline, back-to-catalog link hover.  
**NOT used for:** Primary buttons (these use `bg-text-custom` / `#1a1a1a` — dark), easy/medium badges, form borders, nav active states.

## Interaction Patterns

### Recipe Card
- **Hover:** Border darkens to `--color-text-custom`, card lifts with translate-y + shadow (`hover:shadow-[8px_8px_0px_0px_rgba(0,0,0,0.08)] hover:-translate-y-1`), image scales 105%, title text turns accent red
- **Click:** Navigate to `/recipes/{recipe._id}` (entire card is clickable)
- **Image fallback:** If `imageUrl` fails to load (`onError`), show colored placeholder with first letter of title — background color derived from title hash across a 6-color palette

### Filter Bar (Home page)
- **Search:** Free-text input with search icon (left), debounced at 300ms. On change, updates `search` state → client-side filtering. Syncs to URL query param `?q={value}` via `useSearchParams`
- **Category select:** Dropdown with options: All Categories, Breakfast, Lunch, Dinner, Dessert, Snack, Other. On change, triggers API re-fetch with `?category={value}` query param
- **Difficulty select:** Dropdown with options: All Difficulties, Easy, Medium, Hard. On change, triggers API re-fetch with `?difficulty={value}` query param
- **URL sync:** On mount, read `?q`, `?category`, `?difficulty` from URL to set initial filter state. On filter change, push new params to URL via `useSearchParams` (replace, not push — no history spam)

### Recipe Detail Page
- **Ingredient checklist (UX-07):** Each ingredient row in the table has a checkbox on the left. Checking toggles `line-through decoration-border-custom/50 text-text-muted/60` on the ingredient name and amount. Checked state stored in `useState<Set<number>>` keyed by ingredient index. State is local only — resets on page leave.
- **Star rating (view):** Display-only, filled stars in `--color-star`, empty in `--color-border-custom`
- **Star rating (interactive):** Clickable stars in comment form, hover scale 110%, active scale 95%, numeric indicator shows `{rating} / 5`
- **Comment post:** After successful POST, refetch comments from API (do not manually insert). Reset form fields. Show toast on success/failure via sonner.
- **Comment delete:** Click "Delete" → `window.confirm()` → on confirm, DELETE API call → refetch comments. Show toast on success.
- **Recipe delete:** Click "Delete" → inline confirmation box appears → click "Confirm" → DELETE API call → navigate to `/` on success. Show toast. Show error alert on failure.
- **Back navigation:** "Back to Catalog" link at top (mono uppercase + ArrowLeft icon)

### Loading States
- **Feed loading:** 3 skeleton cards in responsive grid (1/2/3 columns), each with: gray image block (aspect-[16/9]), metadata bar, title bar, divider, time/servings bar, rating bar. Uses `bg-neutral-200 animate-pulse` (or shadcn Skeleton component).
- **Detail loading:** Full two-column skeleton mirroring the real layout — hero image block, title, description, metadata grid, author card, ingredients block, comments section.
- **Comment submission loading:** Button text changes to "Submitting feedback...", button disabled with `cursor-wait` and reduced opacity.

### Empty States
- **No recipes match filters:** Centered message with heading "No recipes found.", body explaining no matches, "Clear Search & Filters" button
- **No comments:** Italic centered message: "No feedback comments left yet. Be the first to make and review this cooking masterpiece!"
- **Unauthenticated comment section:** "You must possess an active account..." message with "Sign in to leave a comment" CTA button linking to `/login`

### Error States
- **Feed fetch error:** Red-bordered alert box with "Error Connecting Server" heading + error message body. Recipes grid hidden.
- **Detail fetch error:** Centered error card with "Unable to Load Recipe" heading + error message + "Return to feed" CTA.
- **Comment post error:** Inline error message in red-bordered box above the form (text-xs, font-mono, danger-colored).
- **Comment delete error:** `alert()` with error message (scaffold pattern — may be replaced with sonner toast in implementation).

### Toast Notifications (sonner)
- **Comment posted:** Success toast with "Comment posted" message
- **Comment deleted:** Success toast with "Comment deleted" message  
- **Recipe deleted:** Success toast before navigating away (if delete happens on detail page)
- **Comment post failed:** Error toast with backend message
- **Comment delete failed:** Error toast with backend message
