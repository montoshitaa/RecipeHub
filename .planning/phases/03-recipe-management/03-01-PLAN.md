**Zod schema (recipeFormSchema):**
- `title`: z.string().min(1, 'Title is required')
- `description`: z.string().min(1, 'Description is required')
- `category`: z.string().min(1, 'Category is required')
- `difficulty`: z.enum(['Easy', 'Medium', 'Hard'])
- `time`: z.coerce.number().int().positive('Cook time must be a positive number')
- `servings`: z.coerce.number().int().positive('Servings must be a positive number')
- `imageUrl`: z.union([z.string().url('Invalid URL'), z.literal('')]).optional()
- `tags`: z.string() (comma-separated input, split to array on submit)
- `ingredients`: z.array(z.object({ name: z.string().min(1, 'Name is required'), amount: z.coerce.number().positive('Amount must be positive'), unit: z.string().min(1, 'Unit is required') })).min(1, 'At least one ingredient is required')
- `steps`: z.array(z.string().min(1, 'Step cannot be empty')).min(1, 'At least one step is required')

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser form input → RHF validation | Untrusted user input: title, description, tags, ingredients, steps |
| RHF validated data → createRecipe | Client-validated but still untrusted; backend validates again |
| createRecipe → backend POST /api/recipes | JWT Bearer token in Authorization header (auth boundary) |
| Backend response → CreateRecipe page | Backend may return error messages rendered in UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | RecipeForm submit handler | mitigate | zod schema validates all fields client-side before submission; backend validateRecipePayload re-validates server-side; JWT Bearer token prevents unauthorized mutation |
| T-03-02 | Information Disclosure | Backend error responses | mitigate | Error messages from backend surfaced via sonner toast only — not stored in DOM. Backend returns generic messages for auth failures (403). |
| T-03-03 | Elevation of Privilege | CreateRecipe page /create route | mitigate | ProtectedRoute wraps /create — unauthenticated users redirected to /login. Backend POST /api/recipes uses `protect` middleware (JWT required). |
| T-03-04 | Spoofing | RecipeForm imageUrl field | accept | URL input field accepts any URL string. Backend stores as-is. XSS risk low — React escapes string content in JSX. Image loading uses `referrerPolicy="no-referrer"`. |
| T-03-05 | Denial of Service | Dynamic ingredient/steps arrays | mitigate | Client-side: useFieldArray limits one row at a time. Backend: no rate limiting currently. Accept for v1 MVP — recipe creation is not a high-frequency action. |
| T-03-SC | Tampering | shadcn form component install (npm) | mitigate | slopcheck + verify shadcn/ui@4.10.0 package legitimacy via npmjs.com. Run `npx shadcn@latest add form` from project directory — shadcn is a dev tool, not a runtime dependency. |
</threat_model>
