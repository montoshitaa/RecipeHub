<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser URL :id param → EditRecipe | User-provided recipe ID, could be tampered to access others' recipes |
| EditRecipe → GET /api/recipes/:id | Fetches recipe — any user can fetch any recipe (public endpoint) |
| EditRecipe → PUT /api/recipes/:id | Updates recipe — backend author check (protect + authorId comparison) |
| EditRecipe → DELETE /api/recipes/:id | Deletes recipe — backend author check (protect + authorId comparison) |
| RecipeDetail → DELETE /api/recipes/:id | Deletes recipe from detail page — same backend author check |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Elevation of Privilege | EditRecipe /edit/:id route | mitigate | ProtectedRoute wraps route — unauthenticated users redirected to /login. Client-side check: if recipe.authorId !== user._id, redirect to /unauthorized. Backend PUT/DELETE both check authorId and return 403 if mismatch. |
| T-03-11 | Elevation of Privilege | RecipeDetail delete button | mitigate | Delete button only rendered when `isRecipeOwner` (user._id === recipe.authorId). Backend DELETE checks authorId independently — defense in depth. |
| T-03-12 | Tampering | RecipeDetail delete URL manipulation | mitigate | User could craft a DELETE request to any recipe ID. Backend author check (recipe.authorId !== req.user._id → 403) prevents unauthorized deletion. |
| T-03-13 | Information Disclosure | Error messages on unauthorized edit | mitigate | Client-side redirects to /unauthorized without showing recipe data. Backend returns generic 403 `{ message: 'Only the recipe author can update/delete this recipe' }`. |
| T-03-14 | Repudiation | Recipe deletion (no undo) | mitigate | Confirmation dialog requires explicit user action ("Are you sure?" → "Confirm Delete"). Sonner toast confirms deletion. No undo — this is by design per requirement. |
| T-03-SC | Tampering | npm packages (no new installs) | accept | No new npm packages installed in this plan. updateRecipe/deleteRecipe use existing axios client. |
</threat_model>
